  • Condicionales

    If
    • Condicional "si..."
    • AND con && (tienen que cumplirse ambas)
    • OR con || (solo una condición necesita ser verdadera para que se cumpla)
    • Ej: if (miVariable > 6 && variableNombre == "David") {
      console.log("Variable mayor a 6");
      }
    else if
    • Combinación de If y else
    • Luego de un if
    • "Agarra lo que pasó por el If y lo pasa por otra pregunta"
    • Se puede tener la cantidad de if ese que querramos pero pasa eso es mejor un Switch
    • Ej:
      if (miVariable < 0) {
      console.log("Numero positivo");
      } else if (miVariable === 0) {
      console.log("Numero es cero");
      } else {
      console.log("Numero negativo");
      }
    else
    • va con el If (aunque el condiciona podría tener solo un if y si no se cumple pasa de largo)
    • Ej: if (miVariable <= 6) {
      console.log("Variable menor o igual a 6");
      } else {
      console.log("Variable mayor a 6");
      }
    Switch case
  • Iteradores, Bucles, Ciclos (Loops)

    Incrementación & decrementación: cuando hago algo como i++; eso es igual a i = i + 1; (decrementación es i--;)
    For
    • Usualmente para cuando sabemos la cantidad de iteraciones
    • Ojo: La división de las partes de un for se hace con puntoycoma (;) a diferencia del while que se hace con coma (,).
    • Ej 1: while (variable inicial; condición no alcanzada; incrementar o decrementar) {
      hacer esto;
      }
    • Ej 2:
      let contadorTest = 0; for (contadorTest; contadorTest < 5; contadorTest++) {
      console.log("test ahora es: " + contadorTest);
      }
      console.log("Test final: " + contadorTest);
    • Resultado en consola:
      "test ahora es: 0"
      "test ahora es: 1"
      "test ahora es: 2"
      "test ahora es: 3"
      "test ahora es: 4"
      "Test final: 5"

    While
    • Usualmente para cuando no sabemos la cantidad de iteraciones
    • Ojo: La división de las partes de un while se hace con coma (,) a diferencia del for que se hace con puntoycoma (;).
    • Ej 1: while (variable inicial, condición no alcanzada) {
      hacer esto;
      incrementar o decrementar valor de variable hacia condición;
      }
    • Ej 2:
      let contador = 0;
      while (contador < 5) {
      console.log("Mi contador ahora es: " + contador);
      contador++;
      }
      console.log("Contador final: " + contador);
    • Ojo: En el ejemplo anterior el valor final de la variable "contador" es 5 (ya que la ultima vez - 4 - se fijó si era menor a 5 y estonces corrió el loop y la sumó)
    • Resultado en consola:
      "Mi contador ahora es: 0"
      "Mi contador ahora es: 1"
      "Mi contador ahora es: 2"
      "Mi contador ahora es: 3"
      "Mi contador ahora es: 4"
      "Contador final: 5"

    do... while
    • Va a correr el loop al menos una vez y luego chequea si se cumple la condición

  • Funciones

    Una función nos permite tener un bloque de código que hace algo y reutilizarlo después, no repetir código.
    • Ej 1:
      function saludar() {
      console.log("Hola mundo");
      }
      y luego la llamo para que se ejecute:
      saludar();
    • Ej 2: (función con argumentos)
      function saludar(nombre, edad) {
      console.log("Hola mi nobre es " + nombre);
      console.log("y mi edad es " + edad);
      }
      y luego la llamo para que se ejecute:
      saludar("Laura", 25);
    • Ej 3: (función que retorna)
      function suma(a,b) {
      let result = a+b;
      return result;
      }
      let recibidor = suma(2,4); console.log(recibidor);
    • Ej 3.5: (alternativamente)
      function suma(a,b) {
      return a+b;
      }
      console.log(suma(2,4));
      Los return se usan mucho xq no siempre queremos mostrar en nuestra consola el resultado
    • Ej 4:
      document.getElementById('IdDelElemento').onclick = function() {
      document.getElementById('idDeCosaAModificar').innerHTML = "Contenido del HTML (idDeCosaAModificar) reemplazado";
      }
    • Ej 5:
      document.getElementById('IdDelElemento').addEventListener("click", nombreDeFuncion);
      function nombreDeFuncion() {
      document.getElementById('idDeCosaAModificar').style.backgroundColor = '#FF0000';
      }
  • Arrays
    (arreglos)
    • Generalmente se declaran con la palabra const porque son inmutables.
    • Es recomendable tener el mismo tipo de de dato en un array (por ejemplo solamente strings).
    • Cada elemento tiene un índice, comenzando con el cero(0)
    • Ej:
      let myArray = ["a", true, 6];
    • El array anterior lo puedo ver con
      console.log(myArray);

      pero también con loops:
      let myArray = ["a", true, 6];
      for (let arr = 0; arr < myArray.length; arr++) {
      console.log("accediendo a " + arr);
      let mostrar = myArray[arr];
      console.log(mostrar);
      }
      Resultado:
      "accediendo a 0"
      "a"
      "accediendo a 1"
      true
      "accediendo a 2"
      6
  • Objetos

    • Similares a un array pero nos permite hacer mas cosas.
    • Similar a un Datamap en cuanto tiene claves/propiedades y valores
    • Ej 1:
      let testObjeto = {
      nombre: "David",
      edad: 25,
      trabaja: true
      }
      console.log(testObjeto);
    • El valor puede ser otro objeto, o un array de objetos, etc
    • Ej 2: (array de objetos)
      let arrayDeObjetos = [ testObjeto,
      {nombre: "David",
      edad: 25,
      trabaja: true}
      ]
      console.log(arrayDeObjetos);

    Texto de ejemplo (contenido de HTML)

    Texto a cambiar: <p id="demo1">Texto de ejemplo (contenido de HTML)</p>

    Botón "interno": <button type="button" onclick='document.getElementById("demo").innerHTML = "Hemos cambiado el contenido del HTML"'>Click Me!</button>
    • Cuando haga click onclick
    • en el documento document
    • buscá por el ID del elemento (id="demo") getElementById("demo")
    • reemplazá el texto por est otro = "Hemos cambiado el contenido del HTML"

    Para tener el evento no en el HTML sino en un documento separado voy a necesitar capturar el evento del botón.

    Botón "externo" 1: (algo intermedio, funciona si uno tiene el script de JS solo en el head)
    <button type="button" id="boton1" class="shaded" onclick="javascript:funcion1()">Clickeame!</button>
    JS (Botón "externo" 1): function funcion1() {
    console.log("Capturamos el evento click");
    document.getElementById('demo').innerHTML = "Otro cambio al contenido del HTML";
    }

    Una vez que también agrego el tag <script> en el final del <body> puedo tener la funcionalidad correcta

    Botón "externo" 2: (funciona si uno tiene el script de JS al final del body)
    <button type="button" id='boton3' class="shaded">Botón "externo" 2</button>
    JS (Botón "externo" 2):
    document.getElementById('boton3').onclick = function funcion3() {
    console.log("Capturamos el evento click otra vez");
    document.getElementById('demo1').innerHTML = "Contenido del HTML cambiado";
    }
  • Event Listener
    En este caso está afectando al ejemplo de texto "demo1" que tenemos mas arriba.
    Si no tuviesemos getElementById('boton2'). entonces sería un eventListener general y aplicaría cuando yo hago click en cualquier parte de la página. addEventListener does not overwrite existing event handlers, whereas onclick overrides any existing onclick = fn event handlers.
    Botón eventListener: <button type="button" id="boton2">Botón Escuchador</button>
    JS (eventListener):
    document.getElementById('boton2').addEventListener('click', function() {
    document.getElementById('demo1').innerHTML = "cambio el contenido del HTML desde EventListener";
    console.log("Hola Mundo desde EventListener");
    });

    (Pequeña calculadora)
    Declaro un formulario donde ambos números son obligatorios.
    Ejecutar una función que tome ambos valores, realice una suma, ponga el resultado en la variable "resultado" que pomenos dentro del elemento span y mostramos el elemento div.

    El resultado de la suma es

    <form>
    <label for="numero3">Numero 1:
    <input type="number" id="numero3" required>
    </label>

    <label for="numero4">
    Numero 2: <input type="number" id="numero4" required> </label>
    </form>

    <button type="button" id="sumar">Realizar suma</button>

    <div id="resultadoContainer" style="display: none;">
    El resultado de la suma es </span>
    </div>

    document.getElementById('sumar').addEventListener('click', function(){
    let numeroA = document.getElementById('numero3').value;

    console.log("valor de B es "+numeroB);

    let resultado2 = sumar(parseInt(numeroA), parseInt(numeroB)); //Agrego parseInt para convertir un string a un número, sino me concatena los números como si fuesen strings. console.log("resultado de la suma es "+resultado2);

    //metemos el resultado en el span document.getElementById('resultado2').innerHTML = resultado2;

    //alteramos el diplay: none del contenedor del resultado document.getElementById('resultadoContainer').style.display = 'block';
    });
