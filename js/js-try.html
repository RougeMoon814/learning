    (& Data Types)
  • "Best practice" is to use const everywhere, and if your variable is gonna change, change the variable to a let instead. var use is discouraged.
    var -> original only keyword of variables. Greater scope than let. Not recommended.
    let -> modern way of declaring variables. Their scope is more limited than var, for example to a function. Recommended.
    const -> Can't be changed. Must be assigned a value when declared for the first time.

    • Variable Scope
      • Scope refers to the visibility of variables.
      • Variables defined outside of a function block have a global scope (can be seen everywhere in your JavaScript code)
      • Ej: var myGlobar = 10;
        is a global variable.
      • In
        function fun1() {
        oopsGlobal = 5;
        }
        the variable is global since we "forgot" to put the "var" keyword.

        While here
        function fun1() {
        var theGlobal = 5;
        }
        the variable is only scoped to that function.
      • Variables declared within a function as well as the function parameters, have a local scope (only visible from within the function).
      • Ej:
        function myLocalScope(){
        var myVar = 5;
        console.log(myVar);
        }
        will show the data (5).

        While if I write
        myLocalScope ();
        console.log(myVar);
        Is gonna give a "not defined" error, because we tried to access "myVar" outside of the function.
      • It's possible to have local and global variables with the same name.
      • The local variable takes precedence
      • Ej:
        var outerWear = "T-Shirt";

        function myOutfit() {
        var outerWear = "sweater";

        return outerWear;
        }

        console.log(myOutfit());
        Will return the line "sweater"

        But writing console.log("outerWear");
        still gives us "T-Shirt" (the global variable).

  • Cue
    (Stay in Line)
    • [In computer science] is an abstract data structure where items are kept in order.
    • New items are added to the back of the cue and old items are taken from the front of the cue.
    • Ej:
      function nextInLine(arr, item) {
      arr.push(item);
      return arr.shift();
      }

      var testArr = [1, 2, 3, 4, 5];

      console.log("Before:" + JSON.stringify(testArr));

      console.log(nextInLine(testArr, 6));

      console.log("After:" + JSON.stringify(testArr));

      Before: [1, 2, 3, 4, 5];
      After: [2, 3, 4, 5, 6];
