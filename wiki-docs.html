    • Valor alternativo

    • El tamaño del objeto "toma en cuenta" los paddings y margins, etc.

Object Fit
  • None
    (default)
    • Pone las resoluciones originales (image is not resized).
  • Fill
    • Image is resized to fill the given dimension.
    • If necessary, the image will be stretched or squished
  • Contain
    • Se ubica al medio de el contenedor, expandiendose lo mas posible. Dejando aire donde no.
    • keeps its aspect ratio, but is resized to fit within the given dimension
  • Cover
    • La imagen se ajusta al contenedor y recorta lo que sobra
    • keeps its aspect ratio and fills the given dimension. The image will be clipped to fit
  • Scale-Down
    • Se fija cual de los dos Contain o None da un tamaño mas pequeño y elige ese.
    • image is scaled down to the smallest version of "none" or "contain"

Object-position
  • is used to specify how an <img> or <video> should be positioned within its container.
  • Si tuviese una imagen con image-fit: cover; una parte de mi img va a ser "recortada". Con object-position puedo decirle que se enfoque en particular.
  • Left
  • Right
  • Top
  • Bottom
  • Unidades de medida (horizontal, vertical)
    • px
    • rem
    • %
    • (etc)
  • Ej:
    img { width: 200px; height: 300px; object-fit: cover; object-position: 80% 100%; }

Posicion:
    Modifican el flujo del HTML.
    • Static
      • Valor por defecto.
      • El elemento se considera no posicionado
    • Relative
      • Posicion relativa a donde debiese estar posicionado este elemento
      • No parece diferente pero adquiere las propiedades top, left, right, bottom, z-index.
        • Ej: top: 20px; la mueve 20px hacia abajo.
      • Aunque movamos la caja a otro lado el "espacio en blanco reservado en el HTML para esa caja" no va a ser ocupado por otra cosa. Me queda un "agujero" donde deberia estar la caja.
        • Los 20px los cuenta desde su espacio "original" (no desde el inicio del documento, etc).
      • Si ambas "cajas" tienen position: relative; entonces la segunda se posicionara por encima de la primera.
      • top y left tienen prioridad sobre sus contrapartes (right y bottom). Por lo que si agrego ambas propiedades las primeras tendran prioridad.
      • El Z-index ordena los objetos por capa (cual esta "encima" de otro), cuanto mas grande el valor, mas "superior" la capa.
        • Por defecto su valor comienza siendo 0 (cero), conviene que los elementos tengan una gran diferencia de z-indez por si quiero poner algo en medio (minimo 10 niveles).
        • No funciona si la caja no esta posicionada (o tiene position: static;)
        • Tiene problemas con padres e hijos. Los hijos siempre los muestra encima. La unica forma es ponerle a los hijos un z-index de -1 y nada al padre (no ponerle la propiedad).
    • Absolute
      • Posicion relativa al elemento padre que tenga mas cercano. Si no tiene un elemento padre utiliza la etiqueta de < body >
      • Similar a position: relative; pero con 3 diferencias.
        • El espacio de la caja ya no queda "reservado", osea que no queda un espacio en blanco y otros elementos pueden ocuparlo.
        • Aunque trabaje con un elemento en bloque el ancho ahora se va a ajustar al contenido.
        • Si pongo top: 0; va a hacer quue vaya al top del viewport (si no especifico va a guiarse por su contenedor)
        • El ancho va a ajustarse al contenido (si no tengo cosas como height o width especificado)
    • Fixed
      • (como un float-box, el texto se mantiene fijo en pantalla mientras escrolleamos)
      • Parecido a un absolute (no tiene espacio reservado)
      • Para que no nos tape lo de abajo al body le podemos agregar un padding-top: 100px; y al contenedor del fixed agregarle margin: -100px;
      • Como un espejo embrujado, el fondo te sigue hasta que escrolleas afuera del elemento contenedor.
      • Se usa con menus y publicidades.
      • Dentro de la etiqueta en la que esté, va a flotar en una direccion. Podemos alinear elementos uno al lado del otro si es que a todos les asignamos la misma propiedad de float.
      • float: left / right;
    • Sticky
      • Mezcla entre relative y fixed
      • (como un float-box, el texto se mantiene fijo en pantalla mientras escrolleamos. Util ponerlo con top.)
      • Mezcla entre Fixed y Relative.
      • Conserva el espacio en la pagina (como Relative)
      • Mas notorio a la mitad de dos textos, una vez que uno escrollea hacia abajo se "queda pegado". Definir en que momento de la pagina va a quedar fijo.
      • Ej:
        Position: Fixed; y Top: 0;

Display:
  • Elementos como <p> o <div> son elementos de bloque (la mayoria de los elementos de HTML)
  • Elementos como <span>, <img>, y anchor tags (<link>) son elementos de linea
      • Ambos tienen el atributo de display pero distinto. Puedo cambiarlo. Ej:
    • span { display: block; }
    • div { display: inline; }
    • span { display: inline-block; }
      (similar a la propiedad de inline pero le puedo agregar ancho y alto como a los elementos de bloque. Hoy en dia utilizado para hacer links de navegacion).
    Tambien puedo ocultarlos. Ej:
    • span { display: none; }
    • span { visibility: hidden; }

    Tipos:
    • Inline
      • Generalmente se usa para texto
      • Ej: span, img, etc.
    • Block
      • Ej: p, div, etc.
    • Inline-Block
      • Que se comporte como bloque pero que se adapte al espacio (le puedo agregar ancho y alto).

    • Table
      • En desuso (antes se usaba un monton para armar layout, pero ya no)
      • Inline-Table
        • Tabla pero en linea.
      • Table-Cell
      • Table-Row
      • Table-Column

    • List-Item

    • Flex
      • Revolucionó la forma que se hacian los layout de las paginas web.
      • Se comporta como bloque, pero no el contenido de la "caja".
      • Inline-Flex
    • Grid
      • Mas moderno que flex, y permite mas posibilidades. Pero lo mejor es una combinacion con Flex y Grid.
      • Se comporta como bloque, pero no el contenido de la "caja".
      • Inline-Grid

Animation
  • @keyframes
    • Ej1:(from = 0%, to = 100%)

      @keyframes example { from {background-color: red;} to {background-color: yellow;} }

    • Ej2:

      @keyframes example { 0% {background-color:red; left:0px; top:0px;} 25% {background-color:yellow; left:200px; top:0px;} 50% {background-color:blue; left:200px; top:200px;} 75% {background-color:green; left:0px; top:200px;} 100% {background-color:red; left:0px; top:0px;} }

  • animation-name
    • The name of the animation. Connects the @keyframes with the animation atribute
  • animation-duration
    • If not specified, no animation will occur, because the default value is 0 seconds.
    • Something like 4s, etc.
  • animation-delay
    • Delay for the start of the animation
    • Something like 4s, etc.
  • animation-iteration-count
    • A number like 3 means the animation will run 3 times
    • infinite is also valid.
  • animation-direction
    • normal (default)
    • reverse (backwards)
    • alternate (forward then backwards)
    • alternate-reverse (backwards then forward)
  • animation-timing-function
    • ease (slow, fast, slow)
    • linear (constant)
    • ease-in (slow start)
    • ease-out (slow end)
    • ease-in-out (slow start and end)
    • cubic-bezier(n,n,n,n)
  • animation-fill-mode
    • none
      (Animation will not apply any styles to the element before or after it is executing)
    • forwards
      (element will retain the style values that is set by the last keyframe)
    • backwards
      (element will get the style values that is set by the first keyframe)
    • both
      (animation will follow the rules for both forwards and backwards, extending the animation properties in both directions)
  • animation (shorthand)
    • animation: [name] [duration] [delay] [iteration-count] [direction] [timing-function] [fill-mode]
    • Ej:

      div { animation: animation-ej1 3s 2s infinite alternate linear [none]; }

Responsive Concepts

  • Responsive web design makes your web page look good on all devices. It is called responsive web design when you use CSS and HTML to resize, hide, shrink, enlarge, or move the content to make it look good on any screen.
  • Responsive web design uses only HTML and CSS. Responsive web design is not a program or a JavaScript.
  • Breakpoints are the building blocks of responsive design. Use them to control when your layout can be adapted at a particular viewport or device size. (Ej: a large screen is above 992px wide)

Mobile First
  • Is a technique. Means designing for mobile before designing for desktop or any other device (This will make the page display faster on smaller devices).
  • Instead of changing styles when the width gets smaller than 768px, we should change the design when the width gets larger than 768px.
  • Mobile first, responsive design is the goal.
    Bootstrap's CSS aims to apply the bare minimum of styles to make a layout work at the smallest breakpoint, and then layers on styles to adjust that design for larger devices. This optimizes your CSS, improves rendering time, and provides a great experience for your visitors.

Media Queries

  • W3schools Media Queries Examples
  • The @media rule, introduced in CSS2, made it possible to define different style rules for different media types.
    Media queries in CSS3 extended the CSS2 media types idea: Instead of looking for a type of device, they look at the capability of the device.
  • Using media queries are a popular technique for delivering a tailored style sheet to desktops, laptops, tablets, and mobile phones (such as iPhone and Android phones).
    Use media queries to architect your CSS by breakpoint. Media queries are a feature of CSS that allow you to conditionally apply styles based on a set of browser and operating system parameters. We most commonly use min-width in our media queries.
  • Media queries can be used to check many things, such as:
    • width and height of the viewport
    • width and height of the device
    • orientation (is the tablet/phone in landscape or portrait mode?)
      • Ej:
        @media only screen and (orientation: landscape) { body { background-color: lightblue; } }
    • resolution
  • The result of the query is true if the specified media type matches the type of device the document is being displayed on and all expressions in the media query are true.
    When a media query is true, the corresponding style sheet or style rules are applied, following the normal cascading rules.
    Unless you use the not or only operators, the media type is optional and the all type will be implied.
    We can add a breakpoint where certain parts of the design will behave differently on each side of the breakpoint. You can add as many breakpoints as you like.
  • One way to use media queries is to have an alternate CSS section right inside your style sheet.
  • Ej:
    @media not|only mediatype and (expressions) { CSS-Code; }
  • Media Types
    • all
    • print (for printers)
    • screen
    • speech (for screenreaders)
    • You can have different stylesheets for different media:
      <link rel="stylesheet" media="mediatype and|not|only (expressions)" href="print.css">
  • It might seem odd that we have two sets of identical classes, but it gives us the opportunity in HTML, to decide what will happen with the columns at each breakpoint
    • Ej:
      (For desktop) The first and the third section will both span 3 columns each. The middle section will span 6 columns.
      (For tablets) The first section will span 3 columns, the second will span 9, and the third section will be displayed below the first two sections, and it will span 12 columns

      <div class="row">
      <div class="col-3 col-s-3">...</div>
      <div class="col-6 col-s-9">...</div>
      <div class="col-3 col-s-12">...</div>
      </div>

  • Typical Device Breakpoints
    • (Many. To keep things simple you could target five groups:)
    • /* Extra small devices (phones, 576px and down) */
      @media only screen and (max-width: 576px) {...}
    • /* Small devices (portrait tablets and large phones, 576px and up) */
      @media only screen and (min-width: 576px) {...}

      @media only screen and (min-width: 1600px) {...}

    • column

      • space-around (Espacio igual entre las "cajas" y los bordes, aunque los espacios de los limites son la mitad. Similar a un margin: auto;)

Propiedades de los flex-items
  • Margin
    • El margin dentro de un flex-item se comporta distinto.

    • margin: auto; hará que se centre de todas la direcciones.

    • Ej:

    • Podemos combinarlo con flex-wrap: wrap; para que cuando lo achique hasta el min-width que puede la caja me la pase para abajo.

      flex-grow: 1;
      .
      flex-grow: 2;
  • Flex-Basis
    • Como el width pero mas importante.
    • Si tengo ambas propiedades pero con valores distintos se le va a prestar a tencion al Flex-Basis.
    • Recomendable utilizarlo en lugar del width (por otras propiedades que veremos mas adelante).
    • Solo se empieza a repartir el espacio sobrante (con flex-grow) cuando todas las cajas alcancen el espacio asignado.
  • Flex-Shrink
    • Por defecto todas ceden un espacio de "1" de flex-shrink.
    • Pero si a una de las cajas le pongo flex-shrink: 2; esa caja va a ceder el doble de espacio que las otras, para que las demás puedan ajustarse.
      Pero eso si tienen menos que el width definido (Ej: 300px). A ese ancho definido van a "llegar" todas las cajas juntas.
    • Si le pongo flex-shrink: 3; entregaría el triple de espacio, mientras que flex-shrink: 0.5; entrega la mitad, y asi sucesivamente.
    • Si pongo explicitamente flex-shrink: 0; no va a ceder nada de espacio y las que van a tener que ajustarse son als otras cajas.
  • Flex
    • Es una abreviacion. Necesita por lo menos un parametro.
    • Agrupa flex-grow, flex-shrink, y flex-basis
    • Ej:

    • El que tenga el valor mas grande va al final
    • Ej: Si a alguno de los flex-items le doy un order: 4; ese elemento va a ir al final de la fila (o columna).
      O va a ser el primero si estoy utilizando flex-direction: row-reverse;
    • Muy util, especialmente si cuando quiero que se haga responsive las cajas tengan posiciones distintas.

Grid Layout

    Sin embargo, la mejor práctica es utilizar una combinacion de Flexbox y Grid.
  • Estilo de Layout que se realiza a traves de grillas (como una tabla)
  • Es un valor de display.
    Ej: display: grid;
  • (CSS Tricks) Seamless Responsive Photo Grid

Conceptos
  • Grid Container
    • Toda la grilla (toda la tabla)
  • Grid Item
    • Solo los hijos directos de un Grid Container son Grid Items (como en flexbox), no los nietos y demás.
    • Cada una de las partes del container (Grid-container) es un grid-item
    • Pueden estar dentro o fuera de las celdas, puede haber mas de uno en una celda, etc.
  • Grid Cell
    • Comu una celda (cell)
    • Como un Grid Item pero no necesariamente.
      • Podemos tener una tabla con dos columnas y dos filas (osea 4 celdas), pero solo un item adentro.
  • Grid Tracks
    • Dos Tipos: Grid Column y Grid Row
    • Grid Tracks = Filas + Columnas
  • Grid Area
    • Las definimos nosostros, no estan definidas por defecto.
    • Tendria sentido que ocupen mas de una celda
    • Area de filas y columnas consecutiva (no diagonal, ni cosas como "L").
    • Puede ser la enteridad de una fila o un area cuadrada formada por celdas de diferentes filas y columnas
      • Ej: Celda11, celda12, celda21, y celda22, en un grid 8x8 (osea seria el area del cuadrado de las primeras 4 celdas)
    • Ej: grid-template-areas:
      'header header header header header header'
      'left middle middle middle middle right'
      'footer footer footer footer footer footer';
  • Grid Line (Column / Row)
    • La cantidad de lineas incluye los bordes que delimitan la grilla / tabla
    • Cual es la primera linea divisoria de las columnas?
      No la que divide la columna 1 y la columna 2, sino el "borde" de la grilla

Unidades
  • Podemos usar pixeles, rem, etc.
  • "auto"
  • fr
    • Es como una "fraccion del row" (que en el caso de que no tengamos nada más, sería todo el row).

    • Column

Visual order is usually just as important as the source order, particularly when it comes to accessible interfaces, and grid-auto-flow: dense can sometimes cause a mismatch between the visual and source order.

Practica 1 con Grid
(Galeria)
Image grid
  • Let's try something different with a grid of images where we have a big image and a few thumbnails beside it (or under it).
  • We have two grid configurations (big image on top, and big image on the left). All I am doing is this: .grid img:first-child { grid-area: span 3 / span 3; }
  • grid-area is a shorthand property that combines the following properties into a single declaration: grid-row-start, grid-row-end, grid-column-start, grid-column-end
  • Isn't grid-area the property we use to define named areas? Yes, but we could write a whole lot more about grid-area, but in this particular case:
    .grid img:first-child { grid-area: span 3 / span 3; }
    /* ...is equivalent to: */
    .grid img:first-child { grid-row-start: span 3; grid-column-start: span 3; grid-row-end: auto; grid-column-end: auto; }
  • This means that the first image element in the grid needs to span three columns and three rows. Since we didn't define any columns or rows, the browser does it for us. We've essentially placed the first image in the HTML to take up a 3⨉3 grid. That means that any other images will be placed automatically in those same three columns without the need to specify anything new.
  • As a result, the remaining images in the HTML flow right into place using the same three columns and rows. And since the first image takes up all three columns in the first row, the remaining images flow into additional rows that each contain three columns, where each image takes up a single column.
  • For the second grid configuration (the big image on the left), all I've done is change the automatic flow direction using grid-auto-flow: column the same way we did earlier when placing an <aside> element next to a <section>.
  • This forces the browser to create a fourth column it can use to place the remaining images. And since we have three rows, the remaining images get placed inside the same vertical column.
  • We need to add a few properties to the images to make sure they fit nicely inside the grid without any overflow:
    • .grid { display: grid; grid-gap: 10px; }

      /* for the second grid configuration */
      .horizontal { grid-auto-flow: column; }

      /* The large 3⨉3 image */
      .grid img:first-child { grid-area: span 3 / span 3; }

      /* Help prevent stretched or distorted images */
      img { width: 100%; height: 100%; object-fit: cover; }
  • We can easily update the grid to consider more images by adjusting one value. That would be the 3 in the styles for the large image.
  • But we could add a fourth column simply by changing it to 4 instead:
    .grid img:first-child { grid-area: span 4 / span 4; }

    • Dynamics Pattern Layouts:
      (CSS Tricks) Useful :nth-child Recipes
      • We configure each element so that:
        • The first element needs to take two columns and start at column one (grid-column: 1/span 2).
        • The second element is placed in the third column (grid-column-start: 3).
        • The third element is placed at the first column: (grid-column-start: 1).
        • The fourth element takes two columns and starts at the second column: (grid-column: 2/span 2).
      • in CSS:
        .grid { display: grid; grid-auto-columns: 1fr; /* all the columns are equal */ grid-auto-rows: 100px; /* all the rows are equal to 100px */ }
        .grid :nth-child(4n + 1) { grid-column: 1/span 2; }
        .grid :nth-child(4n + 2) { grid-column-start: 3; }
        .grid :nth-child(4n + 3) { grid-column-start: 1; }
        .grid :nth-child(4n + 4) { grid-column: 2/span 2; }
      • We can remove some declarations and rely grid's auto-placement powers. This is the trickiest part and requires a lot of practice to be able to identify what can be removed.
      • We can update grid-column: 1 /span 2 and use only grid-column: span 2
      • By placing the first, second, and fourth items, the grid automatically places the third item in the correct place.
      • We can remove:
        .grid :nth-child(4n + 3) { grid-column-start: 1; }
        and
        .grid :nth-child(4n + 2) { grid-column-start: 3; }
      • If we place the fourth element in the second column while allowing it to take up two full columns, we're forcing the grid to create a third implicit column, giving us a total of three columns without explicitly telling it to. The fourth element cannot go into the first row since the first item is also taking two columns, so it flows to the next row. This configuration leave us with an empty column in the first row and an empty one in the second row.
      • The browser will automatically place the second and third items in those empty spots. So our code becomes even simpler. All it takes is five declarations to create a very cool and very flexible pattern. Why not use grid-template-columns to define explicit columns? We can do that:
        .grid { display: grid; grid-template-columns: repeat(3, 1fr); /* all the columns are equal */ grid-auto-rows: 100px; /* all the rows are equal to 100px */ }
        .grid :nth-child(4n + 1), .grid :nth-child(4n + 4) { grid-column: span 2; }
      • The code is definitely more intuitive. We define three explicit grid columns and we tell the browser that the first and fourth elements need to take two columns. The explicit approach is more straightforward, while an implicit grid requires you to "fill in the gaps" where CSS is doing additional work behind the scenes.

Responsive Media

Responsive Images
  • W3schools Responsive Web Design (Images)
  • Width
    • If the width property is set to a percentage and the height property is set to "auto", the image will be responsive and scale up and down
    • Notice that the image can be scaled up to be larger than its original size. A better solution, in many cases, will be to use the max-width property.
    • min-device-width (instead of min-width) checks the device width, instead of the browser width. Then the image will not change when you resize the browser window
    • Ej:
      img { width: 100%; height: auto; }
  • Background images can also respond to resizing and scaling
    • If the background-size property is set to "contain", the background image will scale. However, the image will keep its aspect ratio.
    • If the background-size property is set to "100% 100%", the background image will stretch.
    • If the background-size property is set to "cover", the background image will scale to cover the entire content area, some part of the background image may be clipped
  • <picture> Element
    • works similar to the <video> and <audio> elements. You set up different sources, and the first source that fits the preferences is the one being used.
    • Ej:
      <picture> <source srcset="img_smallflower.jpg" media="(max-width: 400px)"> <source srcset="img_flowers.jpg"> <img src="img_flowers.jpg" alt="Flowers"> </picture>
    • The srcset attribute is required, and defines the source of the image.
      The media attribute is optional, and accepts the media queries you find in CSS @media rule

Responsive Video
  • W3schools Responsive Web Design (Videos)
  • Width
    • If the width property is set to 100%, the video player will be responsive and scale up and down
    • Notice that the video can be scaled up to be larger than its original size. A better solution, in many cases, will be to use the max-width property.
    • If the max-width property is set to 100%, the video player will scale down if it has to, but never scale up.
    • Ej:
      video { width: 100%; height: auto; }
